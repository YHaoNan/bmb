本软件专注于力量运动，所以无论是在动作开始时、组间休息时，用户总是会将软件置为后台，转而去看看其它应用，刷刷小红书等。

所以，我们希望能追加：
1. 完善的状态机管理。暂无运动 -> 运动中 -> 组间休息中 -> 暂无运动
  - 刚刚进入 处于暂无运动状态
  - 开始任意一个动作，处于运动中状态（同时只能开启一个任务，再点击其它任务的开始按钮时提示）
  - 动作完成，进入组间休息状态
  - 组间休息结束，进入暂无运动状态，组间休息被动作开始打断，直接进入运动中状态
2. 状态转换通知：需要有一个常驻通知栏，它展示着当前的运动状态。通知仅简单分为标题和文字。
  - 暂无运动，标题“尚未开始”，文案就是“当前还没开始训练，抓紧整两下子吧！”
  - 运动中，标题是动作名，第x组 / 计划x组。文案就是 配重 x 次数。努力训练吧~
  - 组间休息中，标题是休息中，文案是“认真回忆动作要领吧“ / ”好好舒展一下吧“ / ”去看看小红薯放松下吧~“。并且组间休息要求通知右侧有一个计时器，是圆形进度条，进度从100到0，中间展示具体秒数
  - 对于这其中可以自行定义的非标准文案，你可以自己生成一些，并且每次随机展示。注意长度应该和我给出的示例差不多。
3. 悬浮窗：处于休息中时，展示悬浮窗计时器（仅当在应用界面外部时，回到应用自动消失），和通知中的圆形进度条保持一致的样式。当休息结束，可以触碰返回到训练页面
4. 震动：当休息结束时，震动提示用户
5. 配置页面添加悬浮窗权限开关
