# 项目介绍
BMB（Big Muscle Builder），中文名大肌霸！

目前旨在帮助我个人以一种方便快捷的方式创建训练计划，执行计划，并方便快捷的记录每次训练的感受。中长期可以有非常简洁直观的看板，看到个人的训练强度、感受的变化，帮助及时调整或看到进步、退步。

# MVP核心功能

- **用户画像**：用户输入自己的身体维度，身高、体重、健身情况
  - 支持修改：支持修改自己的身体维度，健身情况
  - 每一项身体维度的历史值都会被记录，用于绘制折线图
- **创建训练模板**：创建单次训练的模板，可以随意添加动作，每个动作具有重量、次数、休息时间、平替动作。模板具有文件夹，用户可能采用推拉循环或三分化、五分化循环，可能创建这样的文件夹：推日、拉日、胸部...
  - 输入意图，AI创建完整训练模板（使用此功能时若上次更新身体维度健身情况信息较久，需要提示更新，并可以快捷更新），包括动作，以及动作要领，常见错误
  - 基于用户填写的训练模板，AI生成动作要领，常见错误
  - 基于用户填写的训练模板，AI生成评价（比如重复动作太多，训练计划低效）
  - 基于用户填写的训练模板，AI生成平替动作，避免健身房设备不够用的尴尬场面
  - 基于用户填写的训练模板，AI网络搜索可直达的教程，这里需要限定一些优质教程源
  - 任何AI操作不能代替人类，不能卡住人类流程。如训练计划低效，用户仍可以执意创建。
- **开始跟练**：基于某个训练模板开始跟练，此时，用户需要的是快速选中模板，进行每一个动作的严格跟练、打卡。并以低成本、快捷的方式记录运动感受。
  - 考虑到健身房器械占用的问题，用户每一步可以选择训练模板中的任何一个动作（以及平替动作）。
  - 基于次数、休息时间，若用户练多了，或休息时间不够，在开始动作前要警示提醒用户，但不能不让用户开始。需要注意的就是，一个动作可能有若干平替动作，横向上，每一个动作的重量、次数、休息时间设定可能不一样，算法该如何在这种情况下识别并做警示？可能需要设计一个算法。这个你自主设计，我不关心，给我汇报结果即可。
  - 每次动作结束后，用户可以快速点击感受（感受是基于每个动作的）：轻而易举、有点吃力、无法完成。用户可以选择是否出现代偿现象（最好可以选择部位）。并且可以补充训练感受（非必填）。记录感受并非必要操作，用户可能习惯在完成一个动作的所有组后记录（只记录最后一个），或者完成整个训练计划后记录。在用户主动选择退出运动或完成运动之前，都可以记录和修改训练感受。若用户没填写任何感受，退出或保存时要提醒用户。
  - 在一次运动中，用户选择动作后，可以修改重量、次数、休息时间。并且在该次运动中，后续选择这个动作时都将加载用户编辑的值。若用户修改了值，在触发运动完成前，需要提示用户，您已修改xxxx，是否同步到模板。
  - 运动完成后AI总结：只要用户填写了训练感受，在完成后，可选AI总结训练情况。AI会基于训练感受完成总结，并给用户提出建议，如当前计划太轻松了，建议xxxxx，或当前计划对你来说太重了，而且明显出现xxx代偿，建议接下来xxxx。
- **训练看板**：目前还没想好在这个模型下，该如何设计看板。因为模型还是较为松散的。而且我们现在不知道用户实际关注啥。可以大胆的开放性设计。
- **AI统计**：基于跟练记录的完整数据，身体指标数据，进行周统计、月统计。识别实质性进步、反馈到如何修改训练计划等。目前统计需要人工触发。可以大胆的开放性设计。
- **模型管理**：仅支持OpenAI兼容模型，使用langchain作为大模型底层框架。MVP可以做的很简单，只要求填写base_url、model name、apikey就好，不需要复杂建模。
